## How I approach product builds

1. Start with the user decision, not the dataset.
2. Separate facts, signals, assumptions, and unknowns.
3. Build a thin working slice before over-designing.
4. Make data limitations explicit.
5. Prefer explainable decision logic over black-box outputs.
6. Design for auditability when the domain is regulated.
7. Turn each prototype into a clear product case study.
